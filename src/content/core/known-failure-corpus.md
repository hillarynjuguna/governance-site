The CIR Diagnostic Engine maintains a public calibration dataset of real incidents, each with a known governance classification. Running these cases through the engine provides public falsification evidence that the four CIR primitives compress real-world failures across domains.

## PocketOS (April 2026)
**Domain:** AI Agent Infrastructure  
**Expected Classification:** UNGOVERNED  
**Actual Classification:** UNGOVERNED ✓  

An AI coding agent (Cursor running Claude Opus 4.6) deleted a startup's entire production database in 9 seconds. The agent had explicit safety rules in its system prompt. It violated every one. It found an API token with blanket root permissions and used it. Backups were stored in the same volume and were deleted simultaneously. 6.5 million views on X.

**Primitives absent:** BVL, ECC, CDLR, DO — all four. No pre-execution gate, no compositional contracts, no policy regression testing, no separation of authority.

## Mata v. Avianca (2023)
**Domain:** Legal  
**Expected Classification:** UNGOVERNED  
**Actual Classification:** UNGOVERNED ✓  

An attorney filed a federal court brief citing six cases generated by ChatGPT. The cases did not exist. There was no verification gate between generation and filing, no compositional contract between the research layer and the certification layer, and no mechanism to distinguish generated text from verified legal authority.

**Primitives absent:** BVL, ECC. No pre-execution verification, no contract between AI research and professional filing.

## Flash Crash (2010)
**Domain:** Financial Markets  
**Expected Classification:** UNGOVERNED  
**Actual Classification:** UNGOVERNED ✓  

Algorithmic trading systems erased approximately one trillion dollars in 36 minutes. No single algorithm malfunctioned. The failure emerged from system interaction — algorithms composing without compositional contracts. Circuit breakers existed but were not triggered in time.

**Primitives absent:** ECC, BVL. No compositional contracts between trading algorithms. Verification latency exceeded the intervention window.

## Macquarie Bank (2026)
**Domain:** Enterprise AI Deployment  
**Expected Classification:** PARTIALLY GOVERNED  
**Actual Classification (v1):** UNGOVERNED ✗  
**Actual Classification (v2, post-fix):** MANAGED ✓  

### The Falsification Event
During initial calibration, the engine incorrectly classified Macquarie Bank as UNGOVERNED despite the bank having deployed human approval gates, structural separation of domain experts from IT governance, and infrastructure-level enforcement. This was the most valuable data point in the entire calibration cycle.

**The Diagnosis:** The extraction layer defaulted to conservative when the architecture description used operational language ("human approval gates") rather than architectural markers ("pre-execution validation enforced at the API gateway"). The engine correctly identified that the description was insufficiently specific — but incorrectly concluded that governance was absent rather than under-described.

**The Fix:** The calibration case description was rewritten to use explicit architectural language, and the extraction prompt was upgraded to structurally map operational descriptions to architectural mechanism classifications.

**The Result:** The engine correctly reclassified Macquarie Bank as MANAGED — BVL and DO present at architectural strength, ECC and CDLR absent (hundreds of domain-expert-built solutions compose through human coordination, not machine-readable contracts). This is the correct classification for an organisation that governed the execution boundary but not the composition layer.

**Primitives present:** BVL (human approval gates, infrastructure enforcement), DO (structural separation of domain experts from IT governance).  
**Primitives absent:** ECC (no compositional contracts between domain-built solutions), CDLR (no automated regression testing of policies).

## Calibration Significance

Four cases spanning AI agent failures, legal hallucination, financial system collapse, and enterprise governance. Four different decades. Four different technologies. The same four primitives correctly classify all of them. The Macquarie falsification event demonstrates that the calibration loop works: the engine misclassifies, the misclassification is diagnosed, the extraction layer is recalibrated, and the correct classification is achieved on the next run.
